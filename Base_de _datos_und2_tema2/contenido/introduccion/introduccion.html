Sentencias DML y Operaciones CRUD

En el ciclo de vida de cualquier aplicación web, la capacidad de interactuar con la información es el motor principal de su funcionalidad dinámica. Para entenderlo de forma práctica, imaginemos el funcionamiento de una plataforma de pedidos de comida a domicilio.

Las sentencias DDL (Data Definition Language) se encargaron de diseñar la infraestructura de la plataforma: definieron que debe existir una "caja" llamada Pedidos, con espacios específicos para el nombre del plato, el precio y la dirección de entrega. Sin embargo, una aplicación con solo estructuras está vacía; es aquí donde el Lenguaje de Manipulación de Datos (DML) cobra protagonismo, representando el contenido y el movimiento real dentro de esa estructura.

Dominar las operaciones CRUD (Create, Read, Update, Delete) no es solo una tarea técnica, es la base operativa de cualquier negocio digital. En nuestro ejemplo de la App de comidas:

  1. 🆕 Cuando un cliente confirma su carrito, ejecutamos un Create (INSERT) para generar el pedido.
  2. 🛠️ Cuando el repartidor busca qué entregar, la App realiza un Read (SELECT) de los pedidos pendientes.
  3. 🗑️ Si el restaurante termina de cocinar, se hace un Update (UPDATE) para cambiar el estado a "En camino".
  4. 🗑️ Si el usuario cancela antes de ser procesado, se aplica un Delete (DELETE) o un borrado lógico.

✨ Como señalan Silberschatz et al. (2020), el DML permite que el usuario final acceda y transforme la información según sus necesidades (p. 11). Por tanto, para un programador web, el manejo preciso de estas sentencias es lo que permite transformar una base de datos estática en una experiencia de usuario fluida, segura y en tiempo real.
