Sentencias DDL (CREATE, ALTER, DROP)

Imaginen que van a construir una aplicación web, por ejemplo, un blog personal, un carrito de compras o un sistema de tareas. Toda la información dinámica de su sitio (los usuarios, los productos, las publicaciones) necesita un lugar donde ser almacenada de forma organizada y segura para poder ser recuperada y mostrada cuando sea necesario. Ese lugar es la base de datos.

Pero, ¿cómo creamos ese "almacén" digital? No podemos simplemente empezar a tirar información en un cajón sin orden. Necesitamos, primero, construir los estantes, las cajas y definir sus etiquetas. En el mundo de las bases de datos relacionales, esas estructuras se llaman tablas, y el lenguaje que usamos para definirlas, modificarlas o eliminarlas se conoce como DDL (Data Definition Language).

Pensemos en una analogía simple: vamos a construir un armario.

  1. 🆕 CREATE: Es la acción de construir el armario. Decidimos cuántos cajones va a tener (las tablas) y qué forma tendrá cada cajón.
  2. 🛠️ ALTER: Meses después, decidimos que el armario necesita un espejo en la puerta. No tiramos el armario, solo lo modificamos (ALTER) para añadirle una nueva característica.
  3. 🗑️ DROP: Finalmente, si el armario se daña o ya no lo necesitamos, lo eliminamos (DROP) por completo para liberar espacio.

✨ En esta unidad, nos enfocaremos en dominar estas tres operaciones fundamentales. Aprenderemos a crear nuestras tablas con las columnas y tipos de datos correctos, a modificarlas cuando los requisitos de nuestra aplicación web cambien, y a eliminarlas de forma segura cuando ya no sean necesarias. ¡Manos a la obra!
